The one-paragraph verdict

Korg is genuinely impressive solo-built engineering. The core ledger is real and well-tested; the cross-vendor capture and the recall→re-invoke loop are genuinely novel. But the headline claims — “deterministic,” “reversible,” “causally ordered” — outrun the code, and the go-to-market aims at the most crowded market in AI. The fix isn't to build more. It's to pick one identity, make the claims match the code, and re-aim at the lane nobody owns.

seq · 01

What's actually here

Five surfaces, one shared causal ledger. The genuine core is smaller than the marketing — and stronger than its claims.

Korg — the ledger Beta

crates/korg-registry

A real append-only event ledger: monotonic seq_id, triggered_by causal tree, HLC, pure projection folds, tested rewind + checkpoint. This is the genuine core — 21 green tests, atomic writes, mature error handling.

Korg — the runtime Proto→Alpha

crates/korg-runtime · 18.5k LOC

Orchestration monolith: 4-persona swarm, speculative DAG, .ktrans Ed25519 provenance, recovery. The headline machinery and the overclaims both live here.

Adapters + recall + introspect Beta / PoC

adapters/ · recall-mcp · introspect-mcp

Cross-vendor capture + semantic recall + the recall→re-invoke loop. The real commercial wedge. Solid for Claude Code; Codex/Grok are proof-of-concept.

korgex Alpha

~8.4k LOC · Python + VSCode

Provider-agnostic coding agent. Clean loop & rigorous ledger client — but swarm, model-router, sandbox-by-default & self-healing are stubbed or unwired. Reads as a dogfood vehicle.

KorgChat Beta

~3k LOC · Python

Chat on the shared ledger — branching, recall, summarize. Best-tested component (149 tests). A sharp proof of the thesis, but a developer CLI, not a consumer product.

thumper Folded in

korg-runtime scheduler · thump adapter

Absorbed into the runtime's execution scheduler. No longer a standalone surface.

seq · 03

Where the claims outrun the code

An adversarial pass graded the load-bearing claims against the source. A technical audience will check these in an afternoon — so they should be true.

Real / holds Partial Overstated Missing
“The first deterministic cognitive runtime”Overstated
Determinism is confined to the read-side projection folds. The DAG executor runs nothing — it decides success via !command.contains("fail") and signs each run with a fresh random key. “Replay” re-renders a recorded log; it does not re-execute. The cognition (live LLMs), the append path (Utc::now() + Uuid::v4()) are non-deterministic.
“Reversible — Git for cognition (Ctrl-R rewind)”Overstated
rewind() = events.retain(seq ≤ target) — it truncates a log and restores nothing in the real world. Files, shell commands, network calls all persist. A real git-tree restore exists (handle_operator_fork) but is wired to a different path, not Ctrl-R — and it's destructive truncation, not Git's recoverable history.
“Every decision is causally ordered”Overstated
It's a total order by seq_id (single-writer mutex). triggered_by is an advisory parent pointer, not the ordering. The HLC is informational and single-actor. A real vector clock exists — but in the blackboard subsystem, not the ledger.
Cross-vendor capture + semantic recallPartial — holds
Verified working across synthetic Claude/Codex/Grok sessions in one ledger. But only Claude Code has live capture + one-command setup; Codex/Grok are PoC libraries needing manual MITM, and their model-output text isn't even indexed. Recall is O(N) cosine with no disk cache and no recency filter — so “how did I fix this last week?” can't be served.
Tamper-evident JSON ledger (hash chain)Missing
The spec defines ledger_hash / parent_hash for tamper-evidence — neither field exists in the actual EventMetadata. The JSON ledger is not a hash chain; crypto lives only in the separate .ktrans layer. Closing this is the single fix that makes “verifiable” true.
seq · 04

The strategic crux

The wedge you're selling is real — and already taken. The wedge you've actually built is open, and the market is moving toward it.

Selling into — crowded & late

“Cross-vendor local memory you own”

The most contested, best-funded sub-market in AI.

  • Mem0 OpenMemory MCP is a near-verbatim, better-funded, higher-benchmark instance of this exact pitch.
  • Native memory portability shipped industry-wide in a 30-day window (Mar 2026, GDPR Art. 20).
  • Replay/rewind is now table stakes: LangGraph Time-Travel, OpenAI Agents SDK, Temporal ($5B).
  • Korg's retrieval (substring + BGE-small) is commodity — no reranking, no recency.
◷ ~9–12 months late
Should own — open & early

The verifiable cognition ledger

Attributability + reversibility for multi-agent & regulated work.

  • One inspectable local artifact that is crypto-signed + causally-ordered + replayable + workspace-reversible + cross-vendor.
  • No incumbent combines all five. Memory tools store facts; runtimes replay their own graphs only.
  • The MCP 2026 roadmap names governance as THE open gap; tamper-evident agent ledgers are nascent and fragmented.
  • “Memory” becomes one read-model over the ledger — a feature, not the headline.
◷ ~6–9 months early
seq · 05

Next actionable steps

Prioritized. The theme is subtract, don't add. Start with the two starred items — highest impact, lowest effort.

Pick one name & one sentence — put the audit-ledger framing on the front door Impact: HighEffort: LowStart here

“Korg” means five things today (runtime / swarm / SWE-runtime / memory / Obsidian vault). Collapse to one: “Korg — the verifiable cognition ledger. A tamper-evident, replayable, cross-vendor record of everything your agents do.” Demote “cognitive hypervisor” to a how-it-works section.

Implement the ledger_hash / parent_hash chain on EventMetadata Impact: HighEffort: MedStart here

SHA-256 over the canonical prior event so the JSONL itself is independently tamper-evident — instead of relying solely on the separate .ktrans layer. This single change makes “verifiable” literally true.

Wire Ctrl-R rewind to the real git-tree restore Impact: HighEffort: Med

Connect rewind to the handle_operator_fork path (git read-tree --reset -u + git clean) so it restores the workspace, and make it non-destructive by populating branch_id/rewind_of. Then “reversible like Git” is honest.

Make every claim match the code Impact: HighEffort: Low–Med

Implement-or-delete the swarm (pipes prompts to bare python3), model_router (513 LOC dead code), and korgex's 5 phantom tools (advertised, return “Unknown tool”). Kill the “production-grade” overclaim, fix the yvaehkorg.lol vs yaehkorg.lol typo, and retire “gets smarter over time” until reconcile resolves more than 0/50.

Publish the ledger format + korg:introspect@v1 as an open spec Impact: HighEffort: Med

Be the format, not another app. Get observability tools (LangSmith / Langfuse / Phoenix) and memory tools (Mem0 / Zep) to read & write it. Defensibility comes from being the substrate everyone audits against — a race you can win.

Productize the recall → re-invoke loop Impact: MedEffort: Med

“Replay a past successful procedure,” not just recall a fact. Memory-only stores structurally can't do this. Make it your headline demo.

Compose with the durable-execution wave Impact: HighEffort: High

Sit beneath LangGraph / Temporal / OpenAI Agents SDK as the tamper-evident, third-party-verifiable record layer for enterprise audit. That requires the multi-tenant / RBAC / networked story you haven't built — so it's the real roadmap, not the capability-marketplace track.

Demote korgex & KorgChat to reference implementations Impact: MedEffort: Low

As standalone products they fight Claude Code / Cursor / ChatGPT and lose. As proof that one causal substrate hosts both autonomous agent work and interactive chat, they're invaluable. Stop framing korgex as a Cursor competitor.

Bottom line

You've built something most teams of five couldn't. The failure mode isn't capability — it's dispersion and overclaim.

Be the verifiable cognition ledger — the open, tamper-evident, replayable, cross-vendor audit substrate for the agentic era. Make every claim true. Let recall, korgex, korgchat & the runtime be ribs hanging off that one spine.

The market is moving toward you on governance and provenance. Get there first — honestly — and you own a lane nobody else is standing in.
